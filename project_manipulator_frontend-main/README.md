# project_manipulator_frontend


# Как писать игры
в папке ./scripts/games создаёшь файл НазваниеИгры.js

В нём делаешь import:
GameBase from "../GameBase.js"

Создаёшь класс с игрой:
class НазваниеИгры extends GameBase {}

Далее нужно переопределить 4 функции, о них по порядку

## Функции, которые нужно переинициализировать

### 1 initDesks()
эта функция вызывается почти сразу после отрисовки DOM
В ней ты можешь переопределить значения для переменных

this.deskCount = 2
this.deskColumns = 8
this.deskRows = 8

чтобы изменить количество и размеры игровых полей
все поля пока что будут одинаковых размеров

Сразу после этой функции произойдёт отрисовка игровых полей в DOM

### 2 gameInit()
Эта фунция вызовется также в конструкторе после отрисовки игровых полей
Она нужна, чтобы иницизировать переменные для игры (хранение позиций шахматных фигур, хранить какие клетки выбраны пользователем и так далее)
В игре Mover я храню:
* какие клетки выбраны пользователем
* объект картинок для фишек
* массив объектов фишек, каждый объект фишки хранит её позицию (координаты и номер поля и видна ли она на поле)

### 3 gameLogic(row, column, deskNum)
Вызывается каждый раз, когда ползователь нажимает на клетку на игровом поле
И соответственно в фунцию приходят координаты и номер поля клетки, на которую нажали

Тут ты можешь проверить валидность хода и соответстенно подсветить или не подсвечивать клетку, на которую нажали
Можешь подсветить клетки, на которые может походить игрок и многое другое

### 4 onCommandSent(params)
Эта функция вызывается, когда у нас игрок нажмёт кнопку "отправить ход"
и в неё придёт массив из двух значений:
[resultCode, result]
resultCode - код результата отправки данных на сервер
* 0 - успешная отправка
* 1 - команда дошла на бэк, но он ответил ошибкой (невалидный ход, неверный формат данных)
* 2 - команда не долша до бэка (нет интернета, неправильный IP)

result - либо ответ от сервера, либо код ошибки (результат выпонения fetch)

Эта функция нужна, чтобы переместить фишку на поле при успешной отправке команды
или оповестить игрока о том, что произошла сетевая ошибка


## какими функциями ты можешь пользоваться

### setColor(row, column, deskNum, color = this.colors.creamDarker)
Устанвливает клетке поля с заданными координатами параметр 'background-color' со значением color
Нужен, чтобы задать клетке, на которой стоит фишка черный или белый цвет
Либо подсветить поле, которое выбрал игрок

заготовленные цвета для проекта, но ты можешь использовать и свои

colors = {
  accent: '#ADC7A8',
  accentDarker: '#71866d',
  creamDarker: '#d1c4ad',
  black: '#000000',
  white: '#FFFFFF',
}


### setDecoration(row, column, deskNum, decoration = this.decorations.grayPoint)
Устанавливает клетке поля заготовленный эффект
Нужна, чтобы сделать обозначить выбранную фигуру или подсветить доступные ходы
Доступные эффекты (свои не добавить):

decorations = {
  cross - красный крестик
  grayPoint - серый  кружок
  borderAccent - зелёная обводка
  borderCreamDarker - коричневая обводка
}

### setCanSend(flag)
Если передать true - позьзователь сможет отправить команду "походить" и подсветит соответствующую кнопку на экране
Если передать false - наоборот

### setImage(row, column, deskNum, imageLink = '')
Если передать ссылку на изображение - оно установится на клетке
Если передать пустую строку или ничего не передавать - картинка пропадёт
Работает вместе с setColor


## Уточнения по реализации
### Координаты
row: 0 - это нижний ряд на экране игрока
column: 0 - это левый ряд на экране игрока
deskNum - индексирование досок начинается с 0

### Отправка команд
Строку для отправки на бэкенд ты пишешь сам (о синтаксисе договаривайся со своим бэкендером)
Отправляться на бэк будет строка из переменной
this.stringToSend

также ты можешь переопределить адрес отправки команды в переменной
this.url
